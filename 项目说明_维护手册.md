## 二、HTML 数据模型（更新时改这些 JS 常量即可）

| 变量 | 结构 | 含义 |
|---|---|---|
| `T` | `{英文名:[中文,组,FIFA排名,大洲,状态,核心球员,预计首发,分析]}` | 48 队 |
| `H` | `{英文名:[[日期,级别,对手,半场比分,全场比分,结果],...]}` | 历史战绩 |
| `M` | **对象数组**：`{id,t,lvl,grp,h,a,status,score,ht,spf,hcp,official,cs?,h2h,h2hNote,pred,reason}` | 可竞猜/已赛单场(驱动投注详情) |
| `FX` | **数组的数组**(2026-06-14新增)：`[id,日期,星期,北京时间,阶段,组,主队,客队,场馆]` ×104 | **全赛程**(小组72+淘汰32),驱动「赛程比分」页 |
| `CHAMP` | `[[英文名,赔率],...]` ×48 | **冠军盘**(全48队·驱动`champMkt`;来源BetMGM·参考) |
| `F2` | `[[英文名,赔率],...]` ×48(2026-06-14新增) | **冠亚军盘=进决赛两队**(全48队·驱动`f2Mkt`;来源ESPN/DraftKings「to reach Final」·参考;赔率null=待更新如伊朗) |
| `ST` | `{英文名:[赛,胜,平,负,进,失,积分]}` | 积分榜 |
| `NEWS` | `[[日期,分类,标题,来源],...]` | 新闻 |
| `modelCS/modelTG/modelHF(m)` | 函数 | 由**真实让球+胜平负**经泊松模型派生**比分/总进球/半全场参考**赔率 |

**M 单场字段说明（2026-06-13 重构为对象）：**
- `spf`=不让球胜平负 `[主,平,客]`；竞彩本场未开售则为 `null`（界面显示“未开售”）。
- `hcp`=让球胜平负 `{line:"巴西 -1", sup:1.0, o:[主,平,客]}`；`sup`=主队净胜球期望(让球盘口的数值)，模型据此推比分/总进球/半全场。
- `official=true`：胜平负+让球为竞彩官方实测（来源500彩票竞彩，卡片标★官方）。
- `cs`（可选）=官方比分 `[[标签,赔率],...]`，会覆盖模型比分并标★（如巴西vs摩洛哥）。
- `h2h`=双方历史交锋 `[[日期,级别,"A 比分 B"],...]`，空数组表示首次交锋；`h2hNote`=交锋摘要。
- `status`：`buy`(可购买/未赛) → `live`(进行中) → `done`(已结束)；只有 `buy` 进投注区。

**详情页玩法**：① 胜平负 ② 让球胜平负 ③ 比分(31项, 主胜/平/客胜三行) ④ 总进球(0–7+) ⑤ 半全场(9项)。比分/总进球/半全场标“参考·模型”，官方逐项开盘后由定时任务/Chrome 抓 sporttery/500 替换为★官方。

**详情页历史（三段式·按用户要求排序）**：㊀ 本届世界杯战绩(`thisWC`) → ㊁ 双方历史交锋H2H(`h2hTable`，全部) → ㊂ 各自历史战绩(`histTable`，开放押注队近4年目标10场·标赛事等级；未开赛/球队弹窗近5场)。

- 顶部 `updTag` 为时间戳，更新数据后同步修改。

### 体彩投注计算规则（计算器已实现）
- 单注基准 **2 元**。
- 串关(N 串 1)：投注 = 2 × 倍数；奖金 = 2 × 倍数 × ∏(所选赔率)。
- 单关：投注 = 2 × 倍数 × 注数；奖金 = 2 × 倍数 × Σ(所选赔率)。
- 赔率已含返奖率，奖金为税前理论值，实际以体彩官方为准。

## 八、多 Agent 体系与单场数据库（2026-06-14 新增）

**子代理体系（定义在 `子代理体系/`，编排见其 `README_编排.md`，常驻准则见根目录 `CLAUDE.md`）：**

| Agent | 职责 |
|---|---|
| `wc-analysis-agent` | 【分析·全场次综合研判】综合新闻/短视频(抖音等)/资讯/大名单/客观数据,重点结合**本届实际战况**;小组赛=出线+对决,淘汰赛=形势;已完赛只做结果分析(独立不锚定·可与比分相反),未开赛(含未开售)结合既往赛果做赛前分析;产出球队级研判(点击球队用)。**已并入原对阵分析agent** |
| `wc-guess-agent` | 【竞猜·未开赛】**据分析agent结论猜比分**(pred)+派生胜平负/让球/总进球/半全场倾向+价值/凯利;**替换原 `wc-prediction-agent`** |
| `wc-lottery-agent` | 【体彩】竞彩官方赔率全玩法,★官方 vs 参考·模型 |
| `wc-tcofficial-fetch-agent` | 【体彩官方·取数】专职突破 sporttery 反爬/JS 拿到★官方值(冠军48/冠亚军决赛组合50/单场全玩法);方法阶梯:接口→Chrome→聚合站(仅单场)→用户截图;不破解WAF、不编造 |
| `wc-intel-agent` | 【信息】新闻/自媒体/伤停/首发/舆论 |
| `wc-production-agent` | 【制作】整合四方 → 面板/xlsx/JSON/报告 |
| `wc-qa-agent` | 【验收】按rubric质检,不合格令制作agent**重构**后复验(≤3轮) |

流程:`分析agent(全场次研判) → 竞猜agent(据分析猜比分) + 体彩agent(赔率) + 信息agent(软信息) → 制作agent整合 → 验收agent把关 →（通过）交付 /（不通过）打回重构`。
> 2026-06-16 变更:分析agent升级为全场次综合研判(吸收原对阵分析agent);新增竞猜agent替换原预测agent。详见 `子代理体系/README_编排.md`。

**单场比赛数据库 `比赛数据库/`**:每场一个 `<match_id>.json`(模板 `_模板.json`),存 录像/红黄牌/犯规/首发/阵型/客观数据(xG等)/球员评分/分析/赔率/来源。**JSON ↔ xlsx「⑩ 单场详情」↔ 面板 M 三处一致**。录像:`video.local_clips[]` 放下载片段,分析agent用 ffmpeg 抽关键帧佐证(不冒充全场观看)。
