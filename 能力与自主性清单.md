## 🔁 自主性协议（每次 task / 对话开始前默念）

1. **先看本清单**，判断这次有没有更高效的工具/子代理可用，而不是只用最基础的方式。
1.5 **子代理强制自检（每次问答/对话/task）**：开工先问"**这次能不能尽可能多地用子代理**"——能并行分片抓数据就用 `general-purpose`、只读检索用 `Explore`、较大设计用 `Plan`、业务流程按 `子代理体系/` 6 个 agent 分工；能拆就拆、能并行就并行，不要默认单线硬做。
2. **能并行就并行**：大批量抓取（如48队近4年战绩）用子代理分片并行，不要串行硬抓。
3. **抓不到就升级工具**：`web_fetch` 返回空壳/JS 页面 → 改用 Claude in Chrome（渲染 JS）抓真实数据（尤其 sporttery.cn 官方赔率）。
4. **定期复查 MCP 注册表**：是否上线了体育数据/中国体育彩票 connector，有则替换联网抓取。
5. **用完即记**：本次用了/发现了什么有用能力，追加到文末「使用记录」。
6. **不编造**：抓不到标「待更新/参考」。
