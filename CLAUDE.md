## 一、多 Agent 体系(定义见 `子代理体系/`)

| Agent | 文件 | 职责 | 对象 |
|---|---|---|---|
| **分析agent** | `子代理体系/wc-analysis-agent.md` | **全场次综合研判**:综合新闻/短视频/资讯/大名单/客观数据,重点结合**本届实际战况**;小组赛=出线+对决,淘汰赛=形势;已完赛只做结果分析(独立不锚定),未开赛结合既往赛果做赛前分析;产出**球队级研判** | 全部场次+全部球队 |
| **竞猜agent** | `子代理体系/wc-guess-agent.md` | **据分析agent结论猜比分**(pred)+派生胜平负/让球/总进球/半全场倾向+价值/凯利 | 未开赛 buy/soon |
| 体彩agent | `子代理体系/wc-lottery-agent.md` | 竞彩**官方赔率**全玩法,★官方 vs 参考·模型——**官方数据经 Claude in Chrome 调体彩官网获取(见第六节)** | 可押注 buy |
| 组合agent | `子代理体系/wc-combo-agent.md` | **串关组合计算**:按目标收益枚举达标组合,概率排序+分类 | 可押注 buy(派生) |
| 信息agent | `子代理体系/wc-intel-agent.md` | 新闻/自媒体/伤停/首发/舆论 | 全部 |
| 制作agent | `子代理体系/wc-production-agent.md` | 整合 → 面板HTML / xlsx / 单场JSON / 报告docx | 交付物 |
| 验收agent | `子代理体系/wc-qa-agent.md` | 按 rubric 质检,不合格令制作agent**重构**后复验(≤3轮) | 交付物 |

### 编排流程(数据流)
```
全部场次 ─▶ 分析agent ┐(已完赛=结果/未开赛=赛前) 信息agent ─▶(全部场次软信息)
未开赛 ──▶ 竞猜agent ┤(据分析猜比分) ── 五方产出 ─▶ 制作agent ─▶ 交付物 ─▶ 验收agent ─(通过)▶ 交付用户
可押注 ──▶ 体彩agent ┤(官方赔率·Chrome调官网)                       └(不通过·缺陷清单)▶ 制作agent 重构 ↺
可押注 ──▶ 组合agent ┘(读体彩赔率→串关组合)
```
- **执行方式**:当前桌面环境不一定支持自动 spawn 自定义子代理且 `.claude/` 受保护;故主 Claude **按各 agent 文件分段扮演执行**(也可用 `general-purpose` 子代理并行分片)。逻辑、分工、红线以这些文件为准。

## 五、面板接入 AI 悬浮窗(嵌入式 AI 助手)——常驻·必守

> 网页助手 `世界杯AI助手.js`(右下角悬浮窗 + 选中文本唤起 + 对话/执行页面操作;LLM 默认 Puter·GLM 免Key,可切 GLM 官方Key)已接入面板。**任何重写 / 赔率刷新 / 赛后更新面板后,都必须保证它仍在。**

1. **接入方式(唯一一行,勿删)**:`index.html` 的 `</body>` 前必须有且仅有一行
   `<script src="世界杯AI助手.js"></script>`
2. **每次写面板后的收尾校验(必做·幂等)**:确认该行存在且恰好 1 次、文件仍以 `</html>` 收尾;缺失就在 `</body>` 前补回。
   ```bash
   F="index.html"
   grep -q '世界杯AI助手.js' "$F" || awk '/<\/body>/&&!d{print "<script src=\"世界杯AI助手.js\"></script>";d=1}{print}' "$F" > /tmp/p && cp /tmp/p "$F"
   [ "$(grep -c '世界杯AI助手.js' "$F")" = 1 ] && tail -1 "$F" | grep -q '</html>'   # 须为真
   ```
3. **谁要做**:所有会写 `index.html` 的任务都在收尾执行第 2 步。
4. **配合大文件写法**:整文件保存在挂载环境可能截断尾部,故"补回+校验"必须放在**最后一步**;Edit/Write 工具对大文件不可靠,改面板用第七节的 targeted 写法。

## 六、官方数据获取(赔率 + 赛程)= 用 Claude in Chrome 调体彩官网【2026-06-21 起为唯一指定方法】

> **赔率与赛程的"官方"来源,统一改为:用 Claude in Chrome 插件,在体彩官网 sporttery.cn 页面内调用官方 webapi 接口取数。** 详细操作见 `docs/官方数据抓取指南.md`。

**为什么必须走浏览器插件**:`web_fetch` / `WebSearch` 取不到竞彩官方(搜索仅美区、官网接口对服务器直连返回空),`curl/wget/python 抓取被规则禁止`。**唯一可行**=浏览器在用户(国内)电脑上同源调官方接口。

**前提**:用户在 Chrome 安装并连接「Claude in Chrome」插件(`list_connected_browsers` 非空);其网络可访问 sporttery.cn。

**步骤**:
1. `navigate` → `https://www.sporttery.cn/`(取得 tabId)。
2. `javascript_tool` 在页面上下文内 `fetch` 官方接口(同源,CORS 通)。

**① 竞彩足球·五玩法赔率接口**
```
https://webapi.sporttery.cn/gateway/jc/football/getMatchCalculatorV1.qry?poolCode=had,hhad,crs,ttg,hafu&channel=c
```
返回 `value.matchInfoList`(**按日期分组**)→ 每组 `subMatchList` → 每场对象字段:

| 玩法 | 字段 | 结构 | 写入面板 M 的字段 |
|---|---|---|---|
| 胜平负 | `had` | `{h,d,a}`(主/平/客;**无此字段=该场竞彩未开"不让球",属正常**) | `spf:[h,d,a]` 或 `spf:null` |
| 让球胜平负 | `hhad` | `{goalLine,h,d,a}` | `hcp:{line:goalLine,o:[h,d,a]}` |
| 比分 | `crs` | `s0Hs0A`(如 `s01s00`=1:0);`s1sh`=胜其他、`s1sd`=平其他、`s1sa`=负其他 | `cs:[["1:0",odd]…["负其他",odd]]`(31 项) |
| 总进球 | `ttg` | `s0…s7`(0…7+球) | `tg:[["0球",s0]…["7+球",s7]]` |
| 半全场 | `hafu` | `hh,hd,ha,dh,dd,da,ah,ad,aa`(半/全,主队视角) | `hf:[["胜/胜",hh]…["负/负",aa]]` |

球队中文名 `homeTeamAbbName`/`awayTeamAbbName`、`matchDate`、`matchTime`、`matchNumStr`(如"周六036")。写入官方赔率的场次置 `official:true`,面板 `oddsBlocks` 已支持 `tg/hf/cs` 官方渲染并显示「★竞彩官方」。

**② 赛程/在售场次**:同一接口的 `matchInfoList` 即官方**在售场次 + 开赛时间**(这就是赛程的官方口径);全量 104 场赛程见面板 `FX` 数组(已按官方编号)。

**③ 冠军/冠亚军(优胜冠军)**:接口 `…getMatchCalculatorV1.qry?poolCode=cn`;**经测当前可能返回空列表(未挂在售)**。取不到 → 保留海外参考盘并标「参考·非体彩官方」,待竞彩开放再换。

**④ 竞彩售卖规则(判"未开售/在售/已封盘")**:每天 **11:00 开售次三日(t+1、t+2、t+3)**;每天 **22:00(周末 23:00)停售**当日并对**次日(t+1)场次封盘**。等价于:每场 **开盘=赛前第 3 天 11:00、封盘≈赛前一晚 22:00**。**以官方接口 `matchInfoList` 的实际在售列表为最终准绳**(面板 `liveStatus` 已按"开盘 X-3 日 11:00 / 封盘 X-1 日 22:00"实现)。

**红线**:抓到 → 标「★竞彩官方·来源 sporttery·北京时间 hh:mm」;抓不到 → 标「待更新」;**绝不用模型值冒充官方**。

## 七、实战经验与坑(2026-06-21 总结,务必先看)

1. **ID 统一(已修)**:历史上 面板 `M` 数组与全赛程 `FX` 数组用了**两套编号**,赛程页用 `FX` 的 id 去 `M` 取比分 → **张冠李戴**(把别场比分/状态错配)。现已把 `M` 的次轮场次**全部统一为 `FX` 官方编号(一套 ID)**。新增场次必须用 `FX` 里的 `mNN`,不要另起编号。
2. **大文件 / 定时任务损坏面板(已发生过"只剩首页")**:整文件全量重写在挂载环境会**截断尾部**;updTag 等替换若用贪婪正则会**吃掉 `</header>`+导航 `<nav>`+`<main>`+首页 `<section id="bet">`**。**改面板必须**:① 用 targeted 编辑(Python 读全文→精确字符串/锚定正则替换→写临时文件→`os.replace`),**禁止整文件重写**;② 落盘后做**结构自检**(下方),不通过就回滚备份。
   - 结构自检(改完必跑):`<section` 开 == `</section>` 闭(本项目应=8)、`<nav id="nav">`=1、`<main>`=`</main>`=1、`世界杯AI助手.js`=1、文件以 `</html>` 收尾。
3. **liveStatus 完场判定分赛制**:小组赛约 **130 分钟**、淘汰赛约 **210 分钟**(预留加时+点球)后才判"完场待更新",避免长补时/加时被提前判完。
4. **官方数据只能走 Chrome 插件**(见第六节);`web_fetch`/搜索/`curl` 都不行。
5. **改完必用 node 验证**:抽取 `M`/`FX` 跑 `liveStatus`、核对在售/封盘/进行中、断言结构,再交付。
